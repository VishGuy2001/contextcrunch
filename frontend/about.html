Why I built this

A few months into my graduate researcher role, I built ResearchNU — a 7-agent AI research tool that hits 26 live data sources simultaneously. Papers, patents, grants, clinical trials, YC companies. All at once, all free. I kept the whole thing running for $31/month on AWS.

I was proud of that number. Then I looked more carefully at where the costs were going — and at how agents were behaving in long sessions. Context filling up mid-pipeline. Agents re-explaining things to each other that had already been established. Redundant API calls burning tokens I didn't need to burn.

If this was happening to me at $31/month — a solo grad student trying to keep costs near zero — what was happening inside companies spending $85,000/month on AI with nobody watching?

I looked for a tool that would show me my token waste in real time. Count my context fill across Claude, ChatGPT, and Gemini. Tell me which parts of my conversations were redundant. Compress them intelligently using real math. I couldn't find one. So I built ContextCrunch.

The math is real

ContextCrunch uses sentence embeddings and Google's TurboQuant algorithm — presented at ICLR 2026 — to find semantic redundancy and remove it with near-zero meaning loss. Shannon entropy, cosine similarity, product quantization, O(n²) attention complexity. Real math, explained at three levels so anyone can understand it.

The learn pages are interactive — you can see the math working on your own text, with live demos powered by the real backend.

Stack

Python backend (FastAPI + sentence-transformers + TurboQuant) deployed on Google Cloud Run at api.contextcrunch.io. Frontend deployed on Vercel. Open source on GitHub. Free to use, free to deploy.
